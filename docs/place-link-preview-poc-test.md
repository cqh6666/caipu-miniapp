# 分享链接智能添加联调测试指南

## 功能说明

这是 `docs/place-link-preview-product-design.md` 的前后端联调测试指南，用于验证分享链接
智能添加入口、后端预览接口、高德 POI 补全和表单预填链路。

**当前状态：前端已调用真实后端接口**

- 后端接口：`POST /caipu-api/kitchens/{kitchenID}/add-link-previews`
- 打卡地：解析美团 / 大众点评分享文案，调用高德 POI 返回候选。
- 菜谱：复用现有小红书 / B站解析链路，返回添加菜品表单预填字段。
- 高德未配置或查询失败时，打卡地会降级为 `partial`，用基础字段进入手动表单。

## 已实现的功能

### 1. 添加方式选择第一屏（`add-link-preview-panel.vue`）

- ✅ 主入口：点击粘贴分享链接
- ✅ 剪贴板读取与手动输入兜底
- ✅ 内容解析中状态（带动态文案和计时提示）
- ✅ 能力说明卡片（打卡地 / 菜谱灵感）
- ✅ 手动填写信息入口

### 2. 地点候选展示（`place-candidate-sheet-v2.vue`）

- ✅ 候选卡片列表（1-3 个）
- ✅ 推荐标签（高置信候选）
- ✅ 图片展示与占位图降级
- ✅ 评分、人均、匹配理由展示
- ✅ 手动填写兜底入口

### 3. 主页面集成（`pages/index/index.vue`）

- ✅ 打卡点模式下点击"添加打卡点"触发智能识别流程
- ✅ 菜品模式下点击"添加菜品"触发菜谱智能识别流程
- ✅ 解析结果分流（打卡地 / 菜谱）
- ✅ 候选选择后预填现有编辑表单
- ✅ 菜谱识别后预填现有添加菜品表单
- ✅ 手动填写入口回退到传统表单

### 4. 菜谱添加第一屏（`add-recipe-preview-panel.vue`）

- ✅ 主入口：点击粘贴菜谱链接
- ✅ 剪贴板读取与手动输入兜底
- ✅ 内容解析中状态（链接提取 / 平台识别 / 内容获取 / 步骤解析）
- ✅ 平台说明卡片（小红书 / B站）
- ✅ 手动填写菜谱信息入口

```

【地址：顺德区人昌路2号（华美达和悦然里中间停车场）】
【电话：17303028852】@美团 http://dpurl.cn/4zWiEohz
```

**预期结果**：
- 识别为打卡地

- 候选包含：名称、地址、评分、人均、图片、匹配理由
- 当前高德补全下，首个候选应为 `旺记碳烤肥牛(多丰喜市园区北滘店)`

### 测试文案 2：大众点评分享（包含关键词）

```
【彩虹大排档】这家店很不错！大众点评推荐
```

**预期结果**：
- 识别为打卡地
- 展示候选列表

### 测试文案 3：小红书菜谱（触发菜谱流程）

```
粘贴真实小红书菜谱分享链接或 B站视频链接
```

**预期结果**：

**预期结果**：

## 测试步骤

### 步骤 1：启动小程序

```bash
# 方式 1：HBuilderX 编译
npm run wx:auto-preview

# 方式 2：手动
# 在 HBuilderX 中运行到"微信小程序"
```

### 步骤 2：切换到打卡点模式

1. 进入首页
2. 点击顶部 Mode Switcher 的"打卡点"

### 步骤 3：触发智能识别

1. 点击底部"添加打卡点"按钮（或列表为空时的空状态按钮）
2. 应该看到"添加打卡点"弹窗（第一屏）

### 步骤 4：测试剪贴板识别

1. 在手机或模拟器中复制测试文案 1
2. 点击"点击粘贴分享链接"
3. 观察：
   - 显示"内容解析中"状态
   - 文案随阶段变化（提取 → 识别 → 地点 → POI → 整理）
   - 3 秒后显示"可能需要几秒"提示

### 步骤 5：查看候选结果

解析完成后应该看到：
- "找到 3 个可能的地点"标题
- 第一个候选带"推荐"标签
- 每个候选显示：图片、名称、地址、评分、人均、匹配理由
- 底部"没有合适的，手动填写"按钮

### 步骤 6：选择候选

1. 点击任意候选的"使用这个"按钮
2. 应该：
   - 关闭候选列表
   - 打开现有打卡点编辑表单
   - 字段已预填（名称、类型、地址、经纬度、费用、来源、图片）
   - 显示提示："已根据分享链接填入，可继续修改后保存"

### 步骤 7：测试手动填写

在任意阶段点击"手动填写信息"，应该：
- 关闭所有智能识别界面
- 打开传统空白编辑表单

### 步骤 8：测试菜谱添加入口

1. 切换回美食库模式
2. 点击底部"添加菜品"按钮
3. 应该看到"添加菜品"第一屏
4. 粘贴小红书或 B站链接后开始识别
5. 识别完成后应该打开现有添加菜品表单，并预填菜谱名、来源链接和封面图

## 设计亮点验证清单

- [ ] 第一屏布局清晰，主入口突出
- [ ] 解析中状态有适当的等待反馈
- [ ] 候选卡片信息完整，便于对比
- [ ] 推荐标签帮助用户快速决策
- [ ] 匹配理由可解释，增强信任感
- [ ] 手动填写兜底随处可达
- [ ] 预填表单提示明确，用户可控
- [ ] 整体交互流畅，无明显卡顿

## 下一步

当前前后端主链路已经打通，后续增强项：

1. **图片镜像处理**：候选图片保存时转存到 `/uploads/`，避免依赖高德外链。
2. **错误处理增强**：细化网络失败、超时、第三方风控等场景提示。
3. **埋点接入**：记录识别成功率、候选应用率、手动兜底率。
4. **缓存与限流**：为重复分享文案增加短缓存，降低高德接口调用量。

## 已知限制

- 剪贴板读取在真机上需要权限，模拟器可能失败
- 高德图片仍是外链，正式保存前尚未镜像到自有 `/uploads/`
- 高德候选可能出现同商圈、同品牌异地门店，需要用户确认
- 菜谱解析依赖现有小红书 / B站链路的可用性和配置
