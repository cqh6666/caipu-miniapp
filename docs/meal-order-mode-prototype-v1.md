# 点菜模式基础原型（V1）

实测时间：`2026-03-25`  
适用范围：`caipu-miniapp` 当前 `美食库 / 添加 / 空间` 结构

## 1. 产品目标

把“安排某天吃什么”做成类似外卖点单的体验，让情侣可以提前约定菜单并准备食材。

V1 只做以下能力：

- 选择日期点菜
- 从美食库加菜到购物车
- 购物车增删改
- 提交为某天菜单并保存记录

不做：

- 支付、配送、库存
- 多人同时编辑冲突处理
- 复杂订单状态流转

## 2. 核心概念

- `点菜模式`：美食库的一种临时工作状态，目标是给某一天组一张菜单。
- `购物车`：当前日期下的菜单草稿容器。
- `提交菜单`：将购物车草稿保存为“某天菜单记录”。

## 3. 全局交互原则

- 保留底部导航 `美食库 / 添加 / 空间`，防止用户迷路。
- 点菜模式下，`美食库`页视觉上增加明显状态条：`点菜中 · 3月29日 周六`。
- `购物车`提供双入口：右上角购物车图标（快捷）+ 底部悬浮条（主入口，外卖感核心）
- 结算与提交页隐藏底部导航，聚焦完成提交。

## 4. 页面原型（低保真）

### 4.1 首页（入口卡片）

```text
[最近点菜单]
3月29日 周六 · 已选3道
黑椒菠萝牛肉 / 蒜蓉西兰花 / 番茄蛋汤
[去点菜]
```

点击 `去点菜`：

- 若已有当天草稿 -> 直接进入点菜模式
- 若无草稿 -> 先弹日期选择，再进入点菜模式

### 4.2 美食库（点菜模式）

```text
┌ 点菜中 · 3月29日 周六      改日期  退出 ┐
└──────────────────────────┘
[搜索框]
[早餐/正餐筛选] [想吃/吃过筛选]

菜卡 A          [加入菜单] / [- 1 +]
菜卡 B          [加入菜单] / [- 2 +]
菜卡 C          [加入菜单]

（底部悬浮条，固定）
[已选 3 道 | 去结算]
```

交互：

- `加入菜单` 后按钮变为 `- 数量 +`
- 点击菜卡主体仍可进入详情
- 详情页增加按钮：`加入 3月29日 菜单`

### 4.3 购物车抽屉（从右上角或底部条进入）

```text
购物车（3）
黑椒菠萝牛肉   [- 1 +] [删除]
蒜蓉西兰花     [- 1 +] [删除]
番茄蛋汤       [- 1 +] [删除]

备注（可选）
[周六提前备菜]

[清空] [去结算]
```

交互：

- 数量减到 `0` 等于移除该菜
- 空车时 `去结算` 置灰
- `清空` 二次确认

### 4.4 结算提交页（专注流程页）

```text
确认菜单
日期：3月29日 周六（可改）
共 3 道
- 黑椒菠萝牛肉
- 蒜蓉西兰花
- 番茄蛋汤

备注：周六提前备菜

[提交菜单]
```

提交成功后：

- 主按钮：`查看当天菜单`
- 次按钮：`继续点菜`

### 4.5 菜单详情页（记录与回看）

```text
3月29日 周六 · 已提交
创建人：Alex
备注：周六提前备菜

早餐：暂无
正餐：黑椒菠萝牛肉 / 蒜蓉西兰花 / 番茄蛋汤

[删除安排] [修改菜单]
```

## 5. 状态机（V1）

- `draft`：草稿中，可增删改
- `submitted`：已提交，可删除，也可“带出为草稿后继续改”

状态流转：

1. 新建日期草稿 -> `draft`
2. 购物车为空且退出 -> 自动删除草稿
3. 点击提交菜单 -> `submitted`
4. 点击“修改菜单” -> 基于当前已提交菜单生成同日期草稿，原已提交菜单保留，直到用户重新提交覆盖

## 6. 关键交互规则

- 同一日期只允许一张菜单草稿/记录。
- 同一道菜可重复加，数量上限建议 `9`。
- 切换日期时：目标日期有草稿则直接加载；无草稿则创建空草稿
- 已提交菜单点“修改菜单”时，若同日期已有草稿，则直接继续那份草稿，不覆盖已有修改
- 点菜模式下切换到底部 `空间` 等页时提示：`菜单草稿已自动保存，确认离开？`

## 7. 文案建议（外卖感）

- 加入按钮：`加入菜单`
- 底部条：`已选 N 道 · 去结算`
- 提交按钮：`提交菜单`
- 成功态：`这天吃什么已安排好`

## 8. 数据模型草案（最小可用）

`meal_plans`

- `id`
- `kitchen_id`
- `plan_date`（YYYY-MM-DD）
- `status`（`draft` / `submitted`）
- `note`
- `created_by`
- `created_at`
- `updated_at`

`meal_plan_items`

- `id`
- `plan_id`
- `recipe_id`
- `quantity`
- `meal_type_snapshot`
- `title_snapshot`
- `image_snapshot`
- `sort`

说明：保留 `snapshot`，避免菜谱改名或删除后历史记录丢信息。

- 首页入口卡片新增在 `pages/index/index.vue`
- 点菜模式主承载仍在 `pages/index/index.vue`（复用美食库列表）
- 详情页快捷加入入口在 `pages/recipe-detail/index.vue`
- 菜单详情当前已独立为 `pages/meal-plan-detail/index.vue`
