# 打卡点编辑字段增强设计

修改时间：2026-06-25 09:28:28 +0800 CST

适用范围：微信小程序打卡点新增 / 编辑表单、打卡点详情页、打卡点列表卡片、Go 后端
`places` 模型、高德 POI 候选预填链路。

## 1. 结论

打卡点编辑采用**渐进式三阶段填充策略**：从高德候选创建时实现**零输入秒速创建**，状态切换时采集**核心体验信息**，后续按需补充**辅助决策信息**。

核心原则：
1. **零输入创建优先** - 从高德候选创建时，自动填充所有可获取字段，一键保存，无需手填
2. **体验时采集优先** - 状态切换到”去过”时弹窗采集重访意愿、推荐菜、到访日期
3. **渐进披露优先** - 辅助信息放入折叠区，避免表单恐惧
4. **智能推断优先** - 从高德数据智能推断类型、标签、商圈等信息

建议第一阶段落地：

1. **从高德自动预填字段**：名称、地址、经纬度、电话、人均消费、外部评分、图片、外部 POI 标识
2. **智能推断字段**：类型（从餐饮服务/景点推断）、标签（从 type 层级提取）、商圈（从 business_area）
3. **体验采集字段**：重访意愿⭐、推荐菜🍴、到访日期📅（状态切换时弹窗）
4. **可选辅助字段**：就餐提示💡、适合场景🏷️、最佳时段⏰、停车/交通🚗（折叠区）
5. **前后端同步落地**：后端先补齐数据模型、迁移、保存与候选预填；前端再接入草稿
   字段、编辑页、详情页、列表卡片和状态切换体验采集弹窗。

## 2. 高德 API 可用字段分析

基于高德 POI Search API 实际返回数据，可获取以下字段：

### 2.1 高质量字段（直接预填）

| 高德字段 | 本地字段 | 质量评估 | 处理策略 |
| --- | --- | --- | --- |
| `name` | `name` | ⭐⭐⭐⭐⭐ 商户官方名称 | 直接填充 |
| `address` | `address` | ⭐⭐⭐⭐⭐ 精确地址 | 直接填充 |
| `location` | `latitude`, `longitude` | ⭐⭐⭐⭐⭐ GPS 坐标 | 拆分后填充 |
| `tel` | `phone` | ⭐⭐⭐⭐⭐ 商户电话 | 格式化展示（173-0302-8852） |
| `biz_ext.cost` | `price` | ⭐⭐⭐⭐ 人均消费 | 转换为 "¥79/人" |
| `biz_ext.rating` | `rating` | ⭐⭐⭐⭐ 外部评分 | 标注"高德评分 4.7"，只读展示 |
| `photos[].url` | `imageUrls` | ⭐⭐⭐⭐ 商户图片 | 先显示外链，后台异步镜像到本地 |
| `id` | `external_poi_id` | ⭐⭐⭐⭐⭐ POI 唯一标识 | 系统字段，用于去重和更新 |
| 固定值 `amap` | `external_provider` | ⭐⭐⭐⭐⭐ 数据来源 | 系统字段 |

### 2.2 智能推断字段（需解析）

| 高德字段 | 本地字段 | 推断规则 | 示例 |
| --- | --- | --- | --- |
| `type` | `type` | 包含"餐饮" → `food`<br>包含"景点\|公园\|博物馆" → `attraction`<br>其他 → `other` | "餐饮服务;中餐厅" → `food` |
| `type` 层级 | `tags` | 拆分多级分类提取标签 | "中餐厅"、"地方风味" |
| `business_area` | `tags` 或地址补充 | 作为商圈标签或地址补充 | "北滘" |
| `adname` | `tags` 或地址补充 | 行政区域补充 | "顺德区" |

### 2.3 高德没有的字段（必须用户填写）

| 字段 | 为什么高德没有 | 采集策略 |
| --- | --- | --- |
| **重访意愿** ⭐ | 个人主观评价 | 状态切到"去过"时弹窗采集，星级或标签 |
| **推荐菜** 🍴 | 只有评论，无结构化菜品 | 状态切到"去过"时弹窗采集，逗号分隔 |
| **到访日期** 📅 | 个人行为记录 | 状态切到"去过"时自动填今天，可修改 |
| **备注** 📝 | 个人记忆和提醒 | 自由输入，可从备注智能提示提取结构化信息 |
| **就餐提示** 💡 | 个人经验（排队/预约） | 折叠区，去过后更容易填 |
| **适合场景** 🏷️ | 主观判断 | 折叠区，预设标签快捷选择 |
| **最佳时段** ⏰ | 个人偏好 | 折叠区，可选填 |
| **就餐/游玩时长** ⏱️ | 个人经验 | 折叠区，快捷选项 |
| **陪同人推荐** 👥 | 主观判断 | 折叠区，快捷标签 |

**数据质量说明**：
- 高德评分和人均消费可能过期，需标注"来自高德"，不做强承诺
- 图片需要异步镜像到本地，确保长期可用
- 外部 POI ID 用于后续"重新匹配地点"功能，可刷新过期信息

## 3. 新增字段分层

### 3.1 高优先级字段（核心体验）

| 字段 | 建议字段名 | 来源 | 用户价值 | 采集时机 |
| --- | --- | --- | --- | --- |
| 重访意愿 ⭐ | `revisit_rating` | 用户评价 | 最直接的质量判断，驱动后续筛选和推荐 | 状态切到”去过”时弹窗 |
| 推荐菜/推荐项 🍴 | `recommended_items_json` | 用户手填 | 决定”去这家吃/玩什么”，复访价值极高 | 状态切到”去过”时弹窗 |
| 到访日期 📅 | `visited_at` | 用户选择 | 回看时间线、记录实际打卡 | 状态切到”去过”时默认今天 |
| 电话 📞 | `phone` | 高德 `tel`，可手填 | 订位、询问营业、确认排队 | 从高德自动填充 |
| 外部来源 | `external_provider` | 高德/其他 | 支撑后续重新匹配和刷新 | 系统字段，自动填充 |
| 外部 POI ID | `external_poi_id` | 高德 `id` | 去重、排查候选、刷新资料 | 系统字段，自动填充 |

**设计要点**：
- **重访意愿**是新增的最重要字段，用星级（1-5星）或快捷标签（非常推荐/还行/不推荐）
- **推荐菜/推荐项**根据类型动态调整名称：餐厅→”推荐菜”，景点→”推荐打卡点”，其他→”推荐项”
- **到访日期**已有后端字段，但编辑表单暂未维护，需在状态切换时显式采集

### 3.2 中优先级字段（辅助决策）

| 字段 | 建议字段名 | 来源 | 用户价值 | 编辑页展示 |
| --- | --- | --- | --- | --- |
| 外部评分 | `rating` | 高德 `biz_ext.rating` | 从高德候选时快速判断质量 | 只读展示，标注”高德评分 4.7” |
| 就餐提示 💡 | `dining_tips` | 用户手填 | 合并排队/预约信息，一次性记录经验 | 更多信息折叠区 |
| 适合场景 🏷️ | `scenes_json` | 用户选择 | 后续按聚餐、约会、带娃等筛选 | 更多信息，预设标签 |
| 最佳时段 ⏰ | `best_time` | 用户手填 | “日落时最美”、”午市性价比高” | 更多信息 |
| 就餐/游玩时长 ⏱️ | `duration` | 用户选择 | 帮助规划时间，快捷选项 | 更多信息 |
| 陪同人推荐 👥 | `companion_tags_json` | 用户选择 | “适合2人”、”适合带娃” | 更多信息，预设标签 |
| 停车/交通 🚗 | `parking_note` | 用户手填 | 降低出行决策成本 | 更多信息 |

**设计要点**：
- 合并原有的”排队/预约信息”为”就餐提示”，降低字段数量
- “营业时间”暂不采集，高德无此数据，用户维护成本高
- 新增”就餐/游玩时长”和”陪同人推荐”，这是高频决策维度
- 所有中优先级字段放入”更多信息”折叠区，默认收起

## 4. 渐进式三阶段填充策略

### 4.1 阶段一：从高德候选创建（零输入）

**目标**：降低记录门槛，一键秒速创建，捕获灵感瞬间

用户选择高德候选 “旺记碳烤肥牛(多丰喜市园区北滘店)” 后：

```
自动填充字段：
✅ 名称: 旺记碳烤肥牛(多丰喜市园区北滘店)
✅ 地址: 人昌路2号(华美达广场旁) · 北滘
✅ 经纬度: 113.218424, 22.927688
✅ 电话: 173-0302-8852
✅ 人均: ¥79/人
✅ 类型: 餐厅 (从”餐饮服务”智能推断)
✅ 高德评分: 4.7
✅ 图片: [3张自动加载，异步镜像到本地]
✅ 推荐标签: #中餐厅 #地方风味 #北滘
✅ 状态: 想去 (默认)
✅ 来源: 保留分享来源（美团 / 大众点评 / 手动）
✅ 外部数据源: 高德
✅ 外部POI ID: B0JUN7FVJK (系统字段)

用户操作：
┌─────────────────────────────┐
│ [预览卡片显示上述信息]        │
│                              │
│ [直接保存] [继续编辑]         │
└─────────────────────────────┘
```

**交互细节**：
- 点击”直接保存” → 无需任何输入，秒速创建完成
- 点击”继续编辑” → 进入编辑页，可补充备注、修改标签等
- 图片处理：先显示高德外链（快速展示），后台异步镜像，镜像完成后自动替换

**字段映射规则**：

| 高德字段 | 本地字段 | 转换规则 |
| --- | --- | --- |
| `name` | `name` | 直接填充 |
| `address` + `business_area` | `address` | 拼接为 “地址 · 商圈” |
| `location` | `latitude`, `longitude` | 拆分填充 |
| `tel` | `phone` | 直接填充 |
| `biz_ext.cost` | `price` | 转换为 “¥79/人” |
| `biz_ext.rating` | `rating` | 保存原始值，展示时标注”高德评分” |
| `photos[].url` | `imageUrls` | 先保存外链，后台异步镜像 |
| `type` | `type` | 智能推断：包含”餐饮”→`food`，包含”景点”→`attraction` |
| `type` 层级 | `tags` | 提取”中餐厅”、”地方风味”等二三级分类 |
| `business_area` / `adname` | `tags` | 补充”北滘”、”顺德区”等地域标签 |
| `id` | `external_poi_id` | 系统字段，用于去重和后续更新 |
| 固定值 `amap` | `external_provider` | 系统字段，标识 POI 数据来源 |

说明：现有 `source` 字段继续表达用户来源入口，如 `meituan`、`dianping`、`manual`、
`other`；不要在 V1 中新增 `source=amap`。高德只写入 `external_provider=amap` 与
`external_poi_id`，避免破坏现有来源筛选和展示语义。

### 4.2 阶段二：状态切换到”去过”（体验采集）

**目标**：在记忆最新鲜、动机最强烈的时刻采集核心体验信息

当用户将状态从”想去”切换到”去过”时，自动弹出体验记录卡片：

```
┌─────────────────────────────┐
│ 🎉 打卡完成！                 │
│                              │
│ 这家店怎么样？                │
│ ⭐⭐⭐⭐⭐ [非常推荐]         │
│ 轻触星星评分或选择快捷标签     │
│                              │
│ 🍴 推荐菜 (可选)              │
│ [碳烤肥牛，烤鸡翅，肥牛饭]    │
│ 逗号分隔，最多12个             │
│                              │
│ 📅 打卡日期                   │
│ [2026-06-24 今天] ▼          │
│                              │
│ [暂时跳过] [保存体验]         │
└─────────────────────────────┘
```

**字段说明**：

| 字段 | 控件 | 默认值 | 验证规则 |
| --- | --- | --- | --- |
| 重访意愿 ⭐ | 星级评分 (1-5星) | 无默认值，鼓励评分 | 必选（或提供”跳过”选项） |
| 推荐菜 🍴 | 标签式输入框 | 空 | 可选，逗号/换行拆分，最多12个，每个24字 |
| 到访日期 📅 | 日期选择器 | 今天 | 不能晚于今天 |

**交互细节**：
- 重访意愿支持两种输入：点击星星（精确评分）或快捷标签（非常推荐/还行/不推荐）
- 推荐菜输入框提示根据类型动态调整：餐厅→”推荐菜”，景点→”推荐打卡点”
- 点击”暂时跳过” → 只切换状态，不保存体验信息（后续可补充）
- 点击”保存体验” → 保存重访意愿、推荐菜、到访日期

**智能提示**：
- 如果备注中检测到”推荐XX”、”必点XX” → 提示”要不要填到推荐菜，更醒目哦”
- 如果备注中检测到电话号码 → 提示”检测到电话，移到电话字段可以直接拨打”

## 5. 页面展示优化

### 5.1 列表卡片 - 快速扫描优化

**设计目标**：在有限空间展示最有决策价值的信息

```
┌──────────────────────────────┐
│ [图片]  鸿鲜记海鲜烤肉         │
│         ⭐⭐⭐⭐⭐ | ¥80/人      │  ← 重访意愿+人均
│         📍 1.2km · 北滘        │  ← 距离+商圈
│         🍴 碳烤肥牛            │  ← 1个推荐菜
│         #聚餐 #烤肉            │  ← 2个场景标签
│         [想去]                 │  ← 状态徽章
└──────────────────────────────┘

已去过的卡片（突出重访意愿）：
┌──────────────────────────────┐
│ ⭐[图片]  旺记碳烤肥牛         │  ← 左上角金色角标
│         ⭐⭐⭐⭐⭐ 非常推荐      │
│         ¥79/人 · 2026-06-24   │
│         🍴 碳烤肥牛、烤鸡翅     │
│         #中餐厅 #地方风味       │
└──────────────────────────────┘
```

**字段展示规则**：

| 信息 | 展示建议 | 状态要求 |
| --- | --- | --- |
| 重访意愿 ⭐ | 去过：大号星星+文字，想去：不展示 | 仅去过 |
| 人均消费 | “¥79/人”，紧凑展示 | 如果有 |
| 外部评分 | 可选展示”高德4.7”，避免和重访意愿混淆 | 想去时可展示 |
| 推荐菜 🍴 | 展示前1-2个，如”碳烤肥牛、烤鸡翅” | 如果有 |
| 到访日期 | 去过时展示短日期”2026-06-24” | 仅去过 |
| 适合场景 | 复用现有标签位，展示前2个 | 如果有 |
| 距离+商圈 | “1.2km · 北滘”，帮助快速定位 | 如果有 |

**视觉优化**：
- 重访意愿5星的卡片：左上角加金色⭐角标，突出”非常推荐”

### 6.4 字段映射关系

**从高德 API 到本地字段**：

```go
// 伪代码示例
func MapAmapPOIToPlace(poi AmapPOI) Place {
    place := Place{
        Name:              poi.Name,
        Address:           formatAddress(poi.Address, poi.BusinessArea),
        Latitude:          extractLatitude(poi.Location),
        Longitude:         extractLongitude(poi.Location),
        Phone:             poi.Tel,
        Price:             formatCost(poi.BizExt.Cost),  // "79.00" → "¥79/人"
        Rating:            poi.BizExt.Rating,             // "4.7"
        Type:              inferType(poi.Type),           // 智能推断
        Tags:              extractTags(poi),              // 从 type 提取
        ImageUrls:         extractPhotos(poi.Photos),     // 外链，异步镜像
        ExternalProvider:  "amap",
        ExternalPOIID:     poi.ID,
        Source:            source,                         // meituan/dianping/manual/other
    }
    return place
}

func inferType(amapType string) string {
    if strings.Contains(amapType, "餐饮") {
        return "food"
    }
    if strings.Contains(amapType, "景点") || strings.Contains(amapType, "公园") {
        return "attraction"
    }
    return "other"
}

func extractTags(poi AmapPOI) []string {
    tags := []string{}
    // 从 type 层级提取："餐饮服务;中餐厅;特色/地方风味餐厅"
    parts := strings.Split(poi.Type, ";")
    if len(parts) > 1 {
        tags = append(tags, parts[1])  // "中餐厅"
    }
    if len(parts) > 2 {
        // "特色/地方风味餐厅" → "地方风味"
        subParts := strings.Split(parts[2], "/")
        tags = append(tags, subParts...)
    }
    // 补充商圈
    if poi.BusinessArea != "" {
        tags = append(tags, poi.BusinessArea)
    }
    return uniqueTags(tags)  // 去重
}

func formatAddress(address, businessArea string) string {
    if businessArea != "" {
        return address + " · " + businessArea
    }
    return address
}

func formatCost(cost string) string {
    if cost == "" {
        return ""
    }
    // "79.00" → "¥79/人"
    amount := strings.TrimSuffix(cost, ".00")
    return "¥" + amount + "/人"
}
```

### 8.4 图片镜像任务

**触发时机**：创建或更新打卡点时，如果 `imageUrls` 包含外链（非本地 `/uploads/` 路径）

当前项目已经在保存前通过后端 `upload.Service.SaveRemoteImage` 做同步镜像。V1 继续复用
该链路即可，不新增异步任务，避免“创建后图片短暂外链、稍后再替换”带来的前端状态同步
复杂度。

后续如需优化为异步，可再设计：
```go
// 伪代码
func MirrorExternalImages(placeID string, imageUrls []string) {
    for _, url := range imageUrls {
        if !isLocalURL(url) {
            localPath := downloadAndSave(url, placeID)
            if localPath != "" {
                // 更新数据库，替换外链为本地路径
                updatePlaceImage(placeID, url, localPath)
            }
        }
    }
}
```

**处理策略**：
- V1 保存前镜像失败时跳过失败图片，不阻塞创建流程
- V2 如果改异步，下载失败可保留原外链作为备选
- 重复 POI ID 的图片可以复用，减少下载

## 9. 兼容性与风险

| 风险 | 说明 | 处理策略 |
| --- | --- | --- |
| 表单过长恐惧 | 新增字段过多会降低保存意愿 | **三阶段渐进填充**：从高德创建零输入，状态切换采集体验，辅助信息折叠 |
| 外部信息过期 | 评分、人均消费可能变化 | 标注”来自高德”，不做强承诺；支持手动覆盖；提供”重新匹配地点”刷新 |
| POI 误匹配 | 高德候选可能同名异店 | 预览卡片展示地址和图片，用户确认后再保存；保留 POI ID 用于后续排查 |
| 旧数据兼容 | 老记录没有新增字段 | 数据库默认值 + 前端 normalize 兜底；旧打卡点不强制要求补充新字段 |
| 搜索噪声 | 新增字段进入搜索后可能召回过宽 | 第一阶段搜索只加入推荐项、场景、电话；权重低于名称和地址 |
| 图片镜像失败 | 高德图片下载失败或过期 | 保留外链作为备选；前端显示时优先本地路径，失败时降级外链；镜像任务失败不阻塞创建 |
| 用户误删系统字段 | 编辑时误删 external_poi_id | 系统字段不在编辑表单展示，只在详情页底部弱展示；”重新匹配地点”更新时保留历史 |
| 重访意愿未填写 | 切到”去过”但跳过体验采集 | 允许跳过，列表和详情页不显示重访意愿；后续可补充编辑 |
| 推荐项语义混淆 | 餐厅/景点/其他的”推荐项”含义不同 | 根据类型动态调整字段名称和提示文案 |
| 备注迁移成本 | 老用户在备注里写了推荐菜、电话 | 智能提示：检测备注中的结构化信息，引导迁移到对应字段 |
