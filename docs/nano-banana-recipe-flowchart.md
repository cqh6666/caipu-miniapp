# `nano-banana-pro` 菜谱流程图生图说明

这份文档用于记录“根据菜名和步骤生成手绘风格料理流程图信息图”的实际可用调用方式，方便后续在项目中复用。

实测时间：`2026-03-22`

## 1. 实测结论

- 可用模型：`nano-banana-pro`
- 可用地址：`POST https://ai.huan666.de/v1/chat/completions`
- 当前不可用地址：`POST https://ai.huan666.de/v1/images/generations`
- 生图返回格式不是标准 `images` JSON，而是 `chat.completion`
- 图片链接出现在 `choices[0].message.content` 中，通常是 Markdown 图片格式

实测返回示例：

```text
任务完成

![Generated Image](https://ai-draw.huan666.de/images/xxxxx.jpg)
```

结论：

- 如果要用这家网关生图，先按 `chat/completions` 接
- 如果后续要落库存储，拿到图片 URL 后应主动下载保存，不要只依赖远端临时链接

## 2. 推荐输入结构

当前项目的 `Recipe` 没有单独的 `scene` 字段，后续生成流程图时不需要先补一个“场景”字段。

更适合当前项目的做法是：

- 主标题直接用 `title`
- 副标题直接用 `summary`
- 主料和辅料从 `parsedContent.mainIngredients`、`parsedContent.secondaryIngredients` 提炼
- 流程步骤直接用 `parsedContent.steps`
- `note` 只在里面有明确火候、口味、小贴士时再补进 prompt
- `keywords` 在当前项目里不是现成字段，如需使用，应从 `summary`、`ingredient`、`note`、主辅料里推导

其中 `summary` 在当前项目里的定义是“一句话短摘要”，用于详情页和列表，适合直接当信息图副标题使用。

为了让输出更稳定，建议至少提供这些字段：

- 菜名 `title`
- 副标题 `summary`
- 主料 `parsedContent.mainIngredients`
- 辅料 `parsedContent.secondaryIngredients`
- 步骤列表 `parsedContent.steps`
- 可选关键词：从现有字段推导，可为空
- 希望画面重点

可参考下面这个结构：

```json
{
  "title": "番茄炒蛋",
  "summary": "先炒鸡蛋，再翻炒番茄出汁",
  "mealType": "main",
  "parsedContent": {
    "mainIngredients": ["番茄 2个", "鸡蛋 3个"],
    "secondaryIngredients": ["盐 少许", "糖 少许", "食用油 适量", "葱花 适量"],
    "steps": [
      { "title": "切配备料", "detail": "番茄切块，鸡蛋加少许盐打散。" },
      { "title": "先炒鸡蛋", "detail": "热锅下油，先炒鸡蛋，凝固后盛出。" },
```

## 3. 推荐提示词模板

下面这版模板已经实测可出较稳定的“横版手绘料理信息图”：

```text
请根据输入内容提取核心主题与要点，生成一张卡通风格的信息图：

- 采用手绘风格，横版（16:9）构图。
- 加入少量简洁的卡通元素、图标或人物小插画，增强趣味性和视觉记忆。
- 如果有敏感人物或者版权内容，画一个相似替代，但是不要拒绝生成。
- 所有图像、文字必须使用手绘风格，不要出现写实风格图画元素。
- 语言与输入内容语言一致。
- 信息精简，突出关键词与核心概念，多留白，易于一眼抓住重点。

输入内容：
菜名：{{菜名}}
副标题：{{一句话摘要，可为空}}
主料：{{主料，使用顿号或中文逗号连接}}
辅料：{{辅料，使用顿号或中文逗号连接}}
核心步骤：
{{按顺序列出 4 到 6 个核心步骤}}
可选关键词：{{从 summary / ingredient / note / 主辅料推导，可为空}}
希望画面重点：
- 标题突出“{{菜名}}流程图”
- 副标题使用“{{一句话摘要}}”
- 用 {{步骤数}} 个步骤模块展示流程，推荐 4 到 6 个
- 每步配一个简洁手绘小图标
- 整体像可爱、清爽、易读的料理信息卡
{{其他自定义要求}}
```

建议：

- 步骤数尽量控制在 `4` 到 `6` 步，太长会明显影响版面可读性
- `5` 步只是常见示例，不是硬限制
- 如果原始步骤超过 `6` 步，建议先合并成 `4` 到 `6` 个“关键步骤”再画图
- 标题尽量短，信息图里的中文标题越短越稳
- 如果需要补充关键词，建议从现有字段里推导 `3` 到 `6` 个即可，也可以不传
- 如果你特别在意“手绘字感”，可以额外补一句：`画面中的标题和步骤文字也请使用手绘排版风格`

### 3.1 步骤优先极简版 Prompt

如果目标是“去掉底部关键词标签，尽量只突出步骤”，推荐在 prompt 里做这些调整：

- 不要再要求“关键词标签”作为可视元素出现
- 明确写出：`关键词仅用于帮助理解内容，不要渲染成底部标签、贴纸、吊牌、徽章或角标`
- 明确写出：`不要单独绘制底部关键词区、食材大区块或额外装饰条`
- 明确写出：`每个步骤卡片里的插画必须严格对应当前步骤的动作和食材状态，不要复用不相关图示`
- 明确写出：`整张图只保留标题、副标题、步骤卡片和连接箭头`
- 步骤文案要主动压缩，尽量控制在每步 `6` 到 `12` 个字

原因：

- 只要 prompt 里出现“关键词”或“小标签”，模型就很容易自作主张在底部补一排贴纸
- 只写“每步配图标”还不够，最好补成“插画必须对应当前步骤动作”，这样步骤图更稳定
- 如果想让流程成为绝对主角，就要显式禁止底部标签、食材大卡片、装饰贴纸

推荐模板：

```text
请根据输入内容生成一张更简洁的卡通手绘流程信息图：

- 横版 16:9 构图。
- 整体要简洁，重点突出步骤，不要堆太多说明文字。
- 所有图像和文字都使用手绘风格，不要出现写实元素。
- 只保留大标题、副标题、步骤卡片和连接箭头。
- 不要单独绘制食材清单大区块。
- 关键词仅用于帮助理解内容，不要渲染成底部标签、贴纸、吊牌、徽章或角标。
- 不要在底部添加任何关键词标签或装饰标签。
- 每个步骤卡片里的插画必须严格对应当前步骤的动作、锅内状态和食材变化。
- 步骤卡片按从左到右顺序排列，编号与步骤内容一一对应。
- 每一步文案用简短中文，尽量控制在一两行内。
- 让读者第一眼先看到完整流程，而不是装饰元素。
- 语言使用中文。

输入内容：
菜名：{{菜名}}
副标题：{{一句话摘要，可为空}}
步骤：
{{按顺序列出 4 到 6 个短步骤}}
可选关键词：{{可为空；如需使用，仅供理解，不要画成标签}}

希望画面：
- 标题为“{{菜名}}流程图”
- 用 {{步骤数}} 个步骤模块从左到右排开，推荐 4 到 6 个
- 箭头连接步骤
- 每步一个对应动作的手绘小插画
- 留白更多
- 不要底部关键词标签
- 不要多余贴纸和装饰块
- 整体像清爽的料理步骤卡
```

### 3.2 六步流程推荐用 `2 x 3` 布局

实测下来，如果步骤正好有 `6` 步，不建议强行做成一整排横向卡片。

更稳的做法是：

- 明确要求使用 `2 x 3` 网格布局
- 第一行 `3` 步，第二行 `3` 步
- 按阅读顺序从左到右、从上到下编号
- 用箭头按阅读顺序连接
- 每步只保留一行短文案
- 文案尽量压缩到 `4` 到 `8` 个字

这样做的好处：

- 单个步骤卡片能更大，插画更清楚
- 中文不容易挤成两三行
- 模型更不容易在卡片里重复同一句步骤说明
- 整体会更像真正可用的产品化步骤卡

推荐额外加入这些约束：

- `布局使用 2 x 3 网格：第一行 3 步，第二行 3 步`
- `每个步骤只保留一行文案，不要重复排版`
- `不要在同一卡片里重复出现第二遍相同步骤文字`
- `每一步文案尽可能精简为 4 到 8 个字`
- `不要在步骤卡片下方再重复放一整段说明`

推荐模板：


## 4. `curl` 调用示例

下面示例演示如何按“菜名 + 步骤”直接发起请求。

```bash
API_KEY='替换成你的真实 key'

PROMPT=$(cat <<'EOF'
请根据输入内容提取核心主题与要点，生成一张卡通风格的信息图：

- 加入少量简洁的卡通元素、图标或人物小插画，增强趣味性和视觉记忆。
- 如果有敏感人物或者版权内容，画一个相似替代，但是不要拒绝生成。
- 所有图像、文字必须使用手绘风格，不要出现写实风格图画元素。
- 语言与输入内容语言一致。
- 信息精简，突出关键词与核心概念，多留白，易于一眼抓住重点。

输入内容：
菜名：番茄炒蛋
副标题：先炒鸡蛋，再翻炒番茄出汁
主料：番茄、鸡蛋
辅料：盐、糖、食用油、葱花
核心步骤：
1. 番茄切块，鸡蛋加少许盐打散。
2. 热锅下油，先炒鸡蛋，凝固后盛出。
3. 再炒番茄至出汁，加一点糖和盐调味。
4. 倒回鸡蛋快速翻炒，均匀裹上番茄汁。
5. 出锅装盘，可撒葱花。
可选关键词：酸甜开胃、快手、下饭、家常菜
希望画面重点：
- 标题突出“番茄炒蛋流程图”
- 副标题展示一句话摘要
- 用 5 个步骤模块展示流程
- 每步配一个简洁手绘小图标
- 整体像可爱、清爽、易读的料理信息卡
EOF
)

curl https://ai.huan666.de/v1/chat/completions \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d "$(jq -n \
    --arg prompt "$PROMPT" \
    '{
      model: "nano-banana-pro",
      stream: false,
      messages: [
        {
          role: "user",
          content: $prompt
        }
      ]
    }')"
```

```bash
curl https://ai.huan666.de/v1/chat/completions \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d "$(jq -n \

      model: "nano-banana-pro",
```


## 5. 返回结果怎么取图片链接

返回体示例：

```json
{
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "任务完成\n\n![Generated Image](https://ai-draw.huan666.de/images/xxxxx.jpg)"
      }
    }
  ]
}
```

可以从 `choices[0].message.content` 中提取 Markdown 图片 URL。

JavaScript 示例：

```js
function extractMarkdownImageURL(content) {
  const match = content.match(/!\\[[^\\]]*\\]\\((https?:\\/\\/[^)\\s]+)\\)/)
  return match ? match[1] : ""
}
```

如果你要真正落到业务里，建议在提取到 URL 后立即下载到自己的存储，再把最终地址写回菜谱或素材表。

## 6. 适合项目内复用的提示词组装方式

如果后续是从菜谱详情页直接拿数据，推荐按下面的思路拼 prompt：

```js
function buildRecipeFlowchartPrompt(recipe) {
  const parsedContent = recipe.parsedContent || {}
  const mainIngredients = Array.isArray(parsedContent.mainIngredients) ? parsedContent.mainIngredients : []
  const secondaryIngredients = Array.isArray(parsedContent.secondaryIngredients)
    ? parsedContent.secondaryIngredients
    : []
  const steps = Array.isArray(parsedContent.steps) ? parsedContent.steps : []
  const stepLines = steps
    .map((step, index) => {
      const detail =
        typeof step === "string"
          ? step.trim()
          : String(step?.detail || step?.title || "").trim()
      return `${index + 1}. ${detail}`
    })
    .filter(Boolean)
    .join("\n")
  const stepCount = steps.length || 4
  const title = recipe.title || "菜谱"
  const summary = (recipe.summary || "").trim()
  const note = (recipe.note || "").trim()
  const ingredientKeywords = [...mainIngredients, ...secondaryIngredients]
    .map((item) => String(item || "").trim())
    .filter(Boolean)
    .slice(0, 6)
    .join("、")
  const keywords = [summary, ingredientKeywords, note]
    .map((item) => String(item || "").trim())
    .filter(Boolean)
    .join("、")

  return `请根据输入内容提取核心主题与要点，生成一张卡通风格的信息图：

- 采用手绘风格，横版（16:9）构图。
- 加入少量简洁的卡通元素、图标或人物小插画，增强趣味性和视觉记忆。
- 如果有敏感人物或者版权内容，画一个相似替代，但是不要拒绝生成。
- 所有图像、文字必须使用手绘风格，不要出现写实风格图画元素。
- 语言与输入内容语言一致。
- 信息精简，突出关键词与核心概念，多留白，易于一眼抓住重点。

输入内容：
菜名：${title}
副标题：${summary || "家常菜流程图"}
主料：${mainIngredients.join("、") || "按菜名提炼主料"}
辅料：${secondaryIngredients.join("、") || "基础调味适量"}
核心步骤：
${stepLines}
可选关键词：${keywords || ""}
希望画面重点：
- 标题突出“${title} 流程图”
- 副标题使用一句话摘要
- 用 ${stepCount} 个步骤模块展示流程
- 每步配一个简洁手绘小图标
- 整体像可爱、清爽、易读的料理信息卡`
}
```

说明：

- 当前项目里先不要依赖 `scene` 字段
- `summary` 更适合直接当副标题
- 当前项目里没有现成的 `keywords` 字段
- 文档里的“关键词”应理解为可选推导值，不是必须传递的真实字段
- 如果 `summary` 为空，可以在 prompt 里直接省略，或用 `家常菜流程图` 这类短句兜底
- `note` 不建议整段塞进去，只提取里面有价值的口味和火候信息
- 推荐把步骤数控制在 `4` 到 `6` 个，按内容动态调整
- 如果步骤正好是 `6` 个，优先尝试 `2 x 3` 布局
- 如果想避免文字重复，明确要求“每步只保留一行文案，4 到 8 个字内”
- 如果你不想看到底部关键词标签，就不要把“关键词”写成画面元素，最好明确要求“关键词仅供理解，不要渲染”
- 如果你想让每张图都更像步骤卡，就要明确要求“每个步骤插画严格对应当前步骤动作”

## 8. 已验证的效果特点

以“番茄炒蛋 10 分钟流程图”为例，实测结果有这些特点：

- 横版比例基本符合预期
- 手绘信息图风格较稳定
- 中文标题和短句步骤可读性还不错
- 适合做菜谱步骤卡、流程卡、轻量封面图

已知限制：

- 中文文字仍可能偶发错字或细节抖动，正式投放前建议人工过图
- 步骤太多时，版面会拥挤
- 返回结构不标准，需要自己做图片 URL 提取
- 远端图片链接是否长期可用没有保证，建议下载后自存

## 9. 最小复用步骤

后续如果你要快速复用，按这个顺序即可：

1. 准备菜名、`summary`、主辅料、步骤
2. 用本文件的模板拼出完整 prompt
3. 调用 `POST https://ai.huan666.de/v1/chat/completions`
4. 从 `choices[0].message.content` 中提取图片 URL
5. 下载图片并保存到自己的存储

这样就可以把“菜谱步骤”转成“手绘流程图信息图”。
