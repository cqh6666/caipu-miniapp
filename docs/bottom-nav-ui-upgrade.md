# 底部 Bar UI 优化评估与实施文档

修改时间：2026-04-29 CST
原型来源：`我们的美食空间/src/components/BottomNav.tsx`
落地范围：`pages/index/index.vue`
目标平台：微信小程序，`uni-app + Vue 3 + uview-plus`

## 1. 结论

本次已完成小程序首页底部 bar 的视觉优化：保留 `美食库 / 添加 / 空间`
三入口和现有事件链路，将原来的全宽固定底栏改为更接近原型的浮动胶囊导航。

落地策略是“只改 UI，不改业务行为”：

- `美食库` 仍触发 `switchSection('library')`。
- 中间按钮仍触发 `openAddSheet`，继续打开添加菜谱弹层。
- `空间` 仍触发 `switchSection('kitchen')`。
- 点菜模式下的离开确认、草稿保存、浮动结算条和进入/离开动画全部保留。

原型中的中间 FAB 是 AI 助手入口，但当前小程序没有对应 AI 助手 sheet。
因此本轮不把 `添加` 改成 AI 入口，避免引入功能变更和验收歧义。

2026-04-29 二次微调时，基于实际预览截图继续收敛左右 tab 的视觉：
移除可见图标底壳，弱化 inactive 颜色，并把胶囊圆角和阴影调得更接近原型。

2026-04-29 追加修正：空间入口从 uview `grid` 图标改为本地
`lucide Grid` 等价 SVG，避免继续显示为旧的四格图标，也避免用 `view`
拼九宫格时出现线宽和对齐不稳定。

2026-04-29 继续微调：底部胶囊背景从 `32rpx` 左右边距收窄到 `46rpx`，
同时降低白底不透明度和 blur 强度，让下方菜品能在毛玻璃后隐约透出。

2026-04-29 追加修正：左右 tab 从底部对齐改为胶囊内垂直居中，并降低
tab 内容最小高度，减少导航内部顶部空白。

2026-04-29 动效修正：中间 FAB 的按压态不再只依赖父级 CSS `:active`，
增加小程序原生 `hover-class="nav-fab--pressed"` 和短暂 `hover-stay-time`，
避免切换 tab 后立即打开添加弹层时按压动效被重渲染打断。

## 4. 代码变动

### 4.1 页面底部留白

文件：`pages/index/index.vue`

- 普通页面内容底部 padding 从 `176rpx` 调整为 `214rpx`。
- 点菜模式内容底部 padding 从 `294rpx` 调整为 `342rpx`。

目的：浮动胶囊导航和中间凸出 FAB 会占用更多视觉高度，列表尾部需要
更大的滚动安全区，避免最后一张菜卡或页脚被遮挡。

### 4.2 点菜模式结算条位置

文件：`pages/index/index.vue`

- `.meal-order-floating` 的底部位置从
  `env(safe-area-inset-bottom) + 128rpx`
  调整为
  `env(safe-area-inset-bottom) + 168rpx`。

目的：让点菜结算条和新底部胶囊导航之间保留足够间距，避免两个浮层贴在
一起或触控区域互相干扰。

### 4.3 底部导航栏体

文件：`pages/index/index.vue`

- `.bottom-nav` 从全宽贴底改为 `left/right: 46rpx` 的浮动形态。
- 新增 `bottom: calc(env(safe-area-inset-bottom) + 20rpx)`。
- 新增 `min-height: 92rpx`、`border-radius: 46rpx`。
- 背景改为低不透明度暖白渐变，并加入 `backdrop-filter: blur(18rpx)`。
- 阴影改为更轻的棕色浮层阴影和内侧高光，让菜品列表能隐约透出。
- 胶囊内部从 `align-items: flex-end` 调整为 `align-items: center`，
  左右 tab 内容从贴底改为垂直居中，减少顶部空隙。

### 4.4 Tab 与 FAB

文件：`pages/index/index.vue`

- 左右 tab 的 icon shell 从可见卡片壳改为透明触控容器，降低“按钮壳”存在感。
- 图标尺寸统一到 `24`，inactive 颜色调整为更浅的 `#b9aea8`。
- 激活态不再依赖图标底壳，只通过图标和文字颜色 `#6b4d3d` 建立状态。
- `空间` 图标不再使用 uview `grid`，改为 `static/icons/space-grid.svg`
  与 `static/icons/space-grid-active.svg` 两份本地 SVG，active / inactive
  颜色继续跟随底部导航状态色。
- 中间 FAB 增加 `hover-class="nav-fab--pressed"`，并保留原有 `:active`
  作为兜底，确保点击添加菜品时的缩放反馈稳定出现。
- 中间 FAB 从 `108rpx` 微调为 `112rpx`，边框从 `10rpx` 收到 `8rpx`。
- 普通模式下中间入口上浮到 `-30rpx`，更接近原型的凸出按钮。
- 点菜模式下 FAB 缩小为 `96rpx`，但继续保留当前模式化反馈。

## 6. 兼容性与风险

| 风险 | 说明 | 处理 |
| --- | --- | --- |
| 毛玻璃兼容性 | 部分低版本安卓微信可能弱化 `backdrop-filter` | 已提供半透明背景作为自然降级 |
| 点菜模式遮挡 | 新底栏更像浮层，可能压到结算条 | 已同步上移 `meal-order-floating` |
| FAB 功能误读 | 原型是 AI 入口，当前产品是添加菜谱 | 本轮保留 `添加` 文案和 `openAddSheet` |
| 视觉高度增加 | 胶囊 + FAB 占用更多底部空间 | 已增加普通/点菜模式内容底部 padding |

## 8. 建议补测

建议在微信开发者工具或真机中重点补测：

- iPhone 带安全区机型底部导航是否不贴边。
- 低端安卓微信中毛玻璃降级后是否仍能读清文字。
- 普通模式滚动到底部时，最后一张菜卡和 `关于我们` 不被遮挡。
- 点菜模式下结算条、底部导航和弹层入口没有互相遮挡。
- 点击 `空间` 离开点菜模式时，原确认弹窗和草稿同步逻辑仍正常。
